# 锦书 · UI 设计规则

> **简洁 · 浪漫 · 古风文案 · 大圆角 · 现代化**  
> 少即是多，留白即情调；色彩与排版服务于「两心相知」的温柔感；界面现代、圆润，文案含蓄古雅。

---

## 一、设计原则

| 原则 | 说明 |
|------|------|
| **简洁** | 界面元素克制，无多余装饰；信息层级清晰，一屏一事。 |
| **浪漫** | 色彩柔和、大圆角、留白充足；避免生硬直角与高对比。 |
| **古风文案** | 界面文字含蓄古雅：君归矣、心诺、赠礼、私语、吾心、云阁等；提示语可短句、文言感，不白话啰嗦。 |
| **大圆角** | 卡片、按钮、弹窗、输入框统一采用大圆角（20–28 dp），整体气质圆润、柔和、现代。 |
| **现代化** | 布局清晰、间距舒适、Material 3 与无衬线字体；不仿古雕花，不堆砌传统元素，以现代载体承载古风意境。 |

---

## 二、语言文字（古风）

### 2.1 用词约定

- **功能命名**：心诺、赠礼、私语、藏心、吾心、云阁、立一诺、添心意、写私语、二人同契、良人、一言、诺言。
- **操作与状态**：确定 / 取消、保存、复为默认云阁、君归矣、两心相知一事一诺。
- **提示与错误**：简短、可带文言感，如「请先登录」→「请先登入」或保留；成功类可「已保存」「已删除」。

### 2.2 语气与长度

- 标题、入口：2–4 字为主（心诺、赠礼、吾心）。
- 说明、提示：一句说完，避免长段；可适当用「君」「良人」「两心」等词增加氛围。
- 不强求通篇文言，以**简洁、含蓄、不白话啰嗦**为准。

---

## 三、色彩规范

### 3.1 主色（浪漫而不艳）

- **Primary**：玫瑰粉 / 豆沙粉  
  - 建议色值：`#E8A0A0` 或 `#D4A5A5`（低饱和度）
- **Primary Container**：主色浅底，约 15% 透明度或 `#FDF2F2`

### 3.2 中性色

- **Surface**：`#FBF8F8`（偏暖白）
- **Surface Container**：`#FFFFFF` / `#FDFCFC`，卡片与浮层
- **On Surface**：主文字 `#3D3636`，次要 `#7A7373`，占位 `#A89E9E`
- **Outline / Divider**：`#E8E2E2` / `#EDE8E8`

### 3.3 功能色

- Success：柔和绿 `#7BA67B`
- Error：柔和红 `#C97A7A`
- 同一屏主色强调不超过 2 处。

---

## 四、排版

- **字体**：系统无衬线（如 PingFang SC、Roboto），保证可读与现代化。
- **字号**：大标题 22–24 sp，页面标题 18–20 sp，卡片/列表标题 16 sp，正文 14–16 sp，辅助 12–13 sp。
- **行高**：正文 ≥ 1.4，多行可 1.5。

---

## 五、间距与圆角（大圆角 · 现代）

### 5.1 间距（8pt 网格）

- 常用：8、12、16、20、24、32、40 dp
- 页面边距：20–24 dp；卡片内边距：20 dp；区块之间：24 dp

### 5.2 大圆角规范（统一圆润、现代）

| 元素 | 圆角 | 说明 |
|------|------|------|
| 卡片、弹窗、BottomSheet | **24 dp** | 主圆角，一眼可辨 |
| 按钮（主/次） | **16–20 dp** | 与卡片协调 |
| 输入框、搜索框 | **16 dp** | 略小于卡片，仍属大圆角 |
| 底部导航栏 | **24 dp**（顶部圆角） | 可选，与卡片统一 |
| 列表项、Chip、小标签 | **12–16 dp** | 不尖不方 |
| 头像 | 圆形 | 50% 圆角 |
| SnackBar / Toast | **16 dp** | 与按钮一致 |

- 全应用优先 **16 / 20 / 24** 三档，避免 4、6、8 等小圆角，形成统一「大圆角、现代化」气质。

---

## 六、组件规范

### 6.1 按钮

- 主按钮：Primary 填充，圆角 **20 dp**，内边距 16×24。
- 次要 / Outline：圆角 **20 dp**，描边 Outline 色。
- 文字按钮：无圆角块，仅文字色 Primary 或 On Surface。

### 6.2 卡片

- 圆角 **24 dp**，elevation 0–1 或细描边，内容区 padding 20。

### 6.3 输入框

- 圆角 **16 dp**，描边 Outline；焦点时主色描边或底边。

### 6.4 底部导航

- 图标 + 文字，选中 Primary，未选中弱化；整体风格简洁现代。

### 6.5 列表

- 列表项最小高度 56 dp；大圆角卡片或 12–16 dp 圆角列表项；分割线用 Outline 或留白替代。

---

## 七、图标与动效

- **图标**：线性或轻填充，线宽一致，风格统一（Material Icons 等）。
- **动效**：200–300 ms，`Curves.easeOut`；点击反馈轻微，不夸张。
- **空状态 / 插图**：简洁线条，色用 Primary 或 On Surface 弱化。

---

## 八、页面层级

1. 背景：Surface  
2. 内容区：卡片 / 列表（Surface Container，大圆角）  
3. 浮层：弹窗、BottomSheet、SnackBar（大圆角 + 阴影/遮罩）  
4. 强调：主按钮或 1 处 Primary

---

## 九、实施检查清单

- [ ] 全局主题：Primary、Surface、大圆角（16/20/24）在 `ThemeData` 统一。
- [ ] 所有卡片、按钮、输入框、弹窗使用大圆角，无 4–8 dp 小圆角。
- [ ] 界面文案使用古风用词（心诺、赠礼、吾心、云阁等），提示简洁含蓄。
- [ ] 无多余装饰、无高饱和色；对比度达标（WCAG AA）。
